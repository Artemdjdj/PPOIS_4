# Лабораторная работа №3

## Вариант: 6

## Цель
Изучить событийно-ориентированное программирование с использованием библиотеки на языке Python

## Задание
Разработать игровое графическое приложение согласно выбранному варианту. При разработке игры необходимо изучить функциональность оригинальной игры и по умолчанию реализовывать правила оригинальной игры, если нет ограничивающих требований в условиях задания.

## Программа

Программа представляет собой 2D приложение. Игра  является аналогом игры **Moorhuhn**.

## Возможности

* Все возможные настройки приложения, например, карта уровней, характеристики сущностей, хранятся во внешних файлах (конфигурациях) в формате json.

* При запуске приложения пользователю показывается меню из следующих команд: “начать игру”, “таблица рекордов”, “справка”, “выход”. При выборе “справка” пользователю показывается информация о правилах игры.

* В игре присутствуют звуки и музыка (фоновая музыка).

* При завершении игры, если результат игры выше, чем первая строчка в таблице рекордов, то выводится диалог с поздравлением и возможностью ввести имя и сохранить в таблице рекордов.

* В игре реализованы анимации.



## Технологии

* Pygame

Структуру проекта можно подробно изучить в файле **structure.txt**

## Описание работы

При запуске программы подльзователя встречает меню, в котором он может выбрать: 
* играть;
* таблица рекордов;
* справка;
* выход.

При выборе **первого** пункта меню, пользователь запускает игру и в течение 90 секунд длится игра.
* Если пользователь набрал очков больше, чем первое метсо в таблице лидеров, то выводится окно, где пользователь можетт ввести свое имя, полсе этого он будет записан в таблицу рекордов.
* Если пользователь набрал очков меньше, чем первое место в таблице лидеров, то выводится окно, где пользователь может увидеть набранное количество очков.

После завершения игры, пользователь попадает в меню.

При выборе **второго** пункта меню, пользователю показывается таблица лидеров. В таблице лидеров показываются 10 пользователей и их рекорды. 

При выборе **третьего** пункта меню, пользователю показываются правила игры.

При выборе **четвертого** пункта меню, программа завершает свое выполнение.
